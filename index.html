Introducción al HTTP

El servicio web o servicio HTTP por sus siglas en inglés, Hypertext transfer Protocol, es un protocolo de transferencia de hipertexto que se utiliza par distribuir y compartir información mediante páginas web.
HTTP es un protocolo que envía la información en texto plano y por ello, no se condidera como un protocolo seguro para intercambiar información, como alternativa se puede utilizar el protocolo HTTPS, que significa HTTP Secure.

El protocolo HTTPS cifra el intercambio de información y garantiza la confidencialidad de los datos transmitidos.
Los servidores web albergan las páginas de hipertexto y utilizan de forma estándar el puerto 80 TCP si se utiliza el protocolo HTTP. Para el protocolo HTTPS el puerto asociado por defecto es el 443 TCP.

El servicio HTTP (Hypertext transfer protocol) o protocolo de transferencia de Hipertexto, nos permite transferir información (páginas web, imágenes, vídeos, documentos,..etc.) entre un cliente o navegador web y un servidor web a través del puerto 80. Seguramente sea el servicio más importante de Internet.
HTTPS (Hypertext transfer protocol secure) es la versión segura del protocolo HTTP y utiliza el puerto 443 y un cifrado basado en SSL/TLS (Secure Socket Layer y su sucesor Transfer Layer Security).

Los clientes, llamados navegadores web, se conectan a los servidores utilizando estos puertos y solicitan al servidor el envío de un recurso, normalmente una página web. El servidor a continuación, envía su respuesta al cliente y este se encarga de darle formato y representarla en pantalla del dispositivo que se esté utilizando, hoy en día se usa tanto ordenadores como tabletas y smartphones para acceder y consultar la web.

Los tipos MIME

Inicialmente las páginas web contenían texto y enlaces (hypertext) por lo que inicialmente el protocolo HTTP te permitía enviar únicamente texto. Actualmente HTTP nos permite enviar todo tipo de documentos gracias a los tipos MIME. Por ello utiliza encabezados que describen el contenido del propio mensaje mediante codificación MIME. Por lo tanto los tipos MIME son unas especificaciones usadas para dar formato a mensajes no ASCII. IANA es el organismo internacional que define los tipos MIME.

Existen diferentes tipos MIME:
text/html, text/css, video/mpeg (un documento html, una hoja de estilos y un vídeo en formato mpeg).
Los tipos MIME se utilizan en los siguientes casos:

1. Para informar al navegador del tipo de datos que esta recibiendo del servidor.

Se define usando: Content-type dependiendo de si el navegador puede interpretarlo puede:

  • visualizar el documento (por ejemplo text/html),
  • llamar a una aplicación externa (application/pdf al descargar un documento pdf lo leemos con un lector de pdf) o
  • preguntar al usuario que hacer si no sabe con que abrir su contenido (aplication/abb).

2. Para permitir o denegar un contenido: Por ejemplo, Allow: application/doc
3. Para encapsular uno o mas objetos dentro del cuerpo del mensajes. El más conocido son los tipos multipart ( multipart/form-data), que se suelen utilizar junto al método POST para hacer un upload de una imágen al servidor.

Funcionamiento del protocolo

1. El usuario introduce la url en el navegador
2. El navegador o cliente http decodifica la información obteniendo el esquema o protocolo, la IP o el nombre del servidor web, puerto, etc.
3. El cliente conecta con el servidor web y le solicita la pagina web. Petición HTTP (Request).
4. El servidor envía la pagina web o devuelve el código de error correspondiente. Respuesta HTTP (Response).
5. Navegador o cliente http interpreta los códigos html recibidos y visualiza el resultado.
6. Se cierra la conexión.

Por lo tanto cuando se establece una conexión HTTP tenemos dos tipos de mensajes entre un cliente y un servidor: una petición por parte del cliente y una respuesta por parte del servidor.
Si el navegador solicita una pagina web con 2 imágenes, van a existir 3 conexiones: Una pára solicitar la página web y dos para las imágenes.

Normalmente, el recurso solicitado consiste en una página web. Las páginas web, escritas en HTML pueden contener tanto hipertexto como otro tipo de recursos multimedia: imágenes, vídeos, ... Como el servidor cierra la conexión tras el envío de la página web, el navegador tendrá que volver a conectarse con el servidor para solicitarle el resto de recursos que contenga la página web. Estableciendo una conexión por cada recurso.

Este modo de funcionamiento es el que establece la versión 1.0 del protocolo HTTP. La versión 1.1 del protocolo permite mantener la conexión abierta, de modo que el cliente le pueda solicitar al servidor el resto de los recursos necesarios par apoder formar la página web completa sin necesidad de volver a tener que abrir más conexiones, acelerándose de este modo la carga de las páginas en los navegadores.

Técnicas para mantener sesiones HTTP

Como se ha comentado, el protocolo HTTP no tiene memoria y una vez que un usuario ha cambiado de página, no hay forma de saber la actividad anterior. Por ello, no permite por sí mismo mantener sesiones de usuario en los sitios web.
Funcionamiento HTTP
Técnicas para mantener sesiones HTTP
Una forma de suplir este inconveniente en el protocolo HTTP consiste en el uso de las Cookies.

Cada vez que visitamos una página, un servidor es el encargado de responder nuestra petición. Cada solicitud que realizamos y la respuesta que recibimos, son totalmente independientes por naturaleza. Sin embargo, el uso de Cookies hace posible la existencia de un estado entre las distintas peticiones que vamos realizando. Es decir, las peticiones HTTP carecen de estado (stateless), pero si se apoyan en el uso de Cookies pueden tener uno (statefull) y con ello modificar lo que los visitantes ven, en función de sus cookies guardadas. cookies y sesiones

Se pueden emplear cookies, entre otras cosas, para:
-Identificar a un usuario durante una o varias sesiones. Por ejemplo, a la hora de realizar compras a través de una tienda web, se almacena su identidad (login y password) como una cookie y se recuerda a lo largo de diferentes visitas qué es lo que lleva almacenado en su cesta de la compra cada usuario.
-Personalizar un sitio web de acuerdo a las preferencias de cada usuario: definir el contenido, apariencia, etc, que queremos que tenga una determinada página en función de las preferencias del usuario que la esté visitando.

La direcciones web

Para navegar por Internet, cuando un cliente solicita una petición a un servidor web, se realiza mediante una dirección web o URL o Uniform Resource Locator. Por ejemplo: http://www.profesordeinformatica.com/html.

Vamos a diferenciar entre URN, URL y URI.

El término URI engloba tanto a las URLs como a las URNs. Dicho de otro modo, una URL es un tipo de URI y un URN también es un tipo de URI.
Cada vez que escribimos una dirección web en el navegador y accedemos a un sitio, estamos utilizando una URI.
Ejemplo de URN Ejemplos de uri y url

Los URI (Identificadores Uniformes de Recursos o Uniform Resource Identifier) son cadenas que indica un recurso en Internet. Puede llevar datos adicionales que indiquen cosas como de qué forma acceder al recurso.
Los URN (Nombres Uniformes de Recursos o Uniform Resource Name) identifican a un recurso en Internet, pero no los localizan.
Los URL (Localizadores Uniformes de Recursos o Uniform Resource Locator) son secuencias de caracteres que se utilizan para localizar recursos en Internet (paginas web, sonidos, vídeos, imágenes, etc).

Comandos HTTP

Cuando se establece una conexión con un servidor web. este se queda esperando a recibir un comando HTTP por parte del cliente. Los comandos más importantes son:
- GET: Le indica al servidor que le envíe un recurso. Es el que usa el navegador para poder obtener las páginas web y el resto de ficheros necesarios para mostrar la página. Cuando el servidor contesta manda un conjunto de datos denominados cabecera con información del recurso solicitado: tamaño, fecha, tipo y a continuación el recurso en sí.

- POST: Lo utiliza el cliente para el envío de información al servidor. Se utiliza frecuentemente para enviarle al servidor los valores de los campos de los formularios que aparecen en las páginas web
